Company Data Intake & Certification Engine

Raw source received

The original Excel stays private. For this demo, we use anonymized extracted JSON.

Structure detected

The workbook structure was analyzed: daily sheets, monthly status sheet, currencies, cash/bank sections, and control totals.

Transactions staged

Rows were converted into normalized staged transactions with date, direction, method, currency, amount, source sheet, and source row.

Mappings suggested

Raw company categories were mapped to standard report categories with confidence scores.

Validation performed

The system checked balances, monthly summary totals, formula risks, missing data, and review issues.

Manual certification required

Only after review should the data enter the official database and become available for chatbot answers.

Staged Transactions

These are anonymized transaction rows extracted from the March workbook. In the final app, this table becomes the staging layer before database approval.

Mapping Review

Mappings convert company language into the app’s standard reporting categories. Future months can reuse approved mappings.

Validation Results

Validation prevents silent errors. The app should never trust the uploaded file blindly.

Manual Review Queue

Manual work should focus on exceptions, not every row. This table shows items that need attention.

Certification step

This simulates approving the import into the official database. In this simple Railway demo, approval is stored only in your browser. Later, this button will write approved data into PostgreSQL.
